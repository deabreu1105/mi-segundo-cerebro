# Metamodelo Centauro

## Definition
El Metamodelo Centauro describe el perfil óptimo del desarrollador de software en la era de la IA: un equilibrio simbiótico donde la automatización asume la velocidad táctica y repetitiva, mientras el humano preserva la estrategia, la arquitectura y la validación del propósito comercial. El nombre "Centauro" evoca la fusión humano-máquina.

## Key Ideas
- La IA asume lo **táctico**: código repetitivo (boilerplate), autocompletado, traducciones de lenguaje a comandos
- El humano retiene lo **estratégico**: diseño de arquitectura, validación de lógica de negocio, decisiones éticas
- El desarrollador evoluciona de "mecanógrafo de código" a **arquitecto supervisor**
- El valor humano reside en descomponer procesos y auditar lógicas algorítmicas
- Sin dominio de [[fundamentos-inmutables]], no es posible ejercer el rol de arquitecto supervisor

## Applications
- Uso de [[github-copilot]] para autocompletado táctico mientras el humano define la arquitectura
- Uso de [[warp]] para comandos CLI con IA, liberando al humano para enfocarse en la lógica
- [[metodo-del-sandwich]]: humano diseña intención → IA genera estructura → humano valida
- Cuadrante de zonas rojas/verdes del [[cuadrante-de-decision-estrategica]]
