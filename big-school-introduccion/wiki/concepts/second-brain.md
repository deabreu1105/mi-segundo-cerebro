# Second Brain (Segundo Cerebro)

## Definition
Sistema de gestión del conocimiento externo que externaliza la carga cognitiva del profesional. Actúa como una extensión de la memoria humana, almacenando snippets de código, decisiones de proyecto, conceptos clave, bugs resueltos y cualquier información valiosa de forma organizada y recuperable.

## Key Ideas
- La memoria humana es **volátil** — confiar solo en ella es un error sistémico
- El Second Brain debe contener: snippets, decisiones de proyecto (ADRs), conceptos clave, bugs resueltos
- Dos enfoques principales: [[obsidian]] (redes de conocimiento) y [[notion]] (bases de datos de proyectos)
- La captura debe ser **sintética**, no mecánica — transcribir sin procesar no sirve
- El [[metodo-cornell]] y el [[bug-journal]] son metodologías de captura de calidad

## Applications
- Usar [[obsidian]] para red de conceptos interconectados (ideal para aprendizaje)
- Usar [[notion]] para gestión de proyectos y colaboración de equipo
- Usar [[notebooklm]] para síntesis de documentación técnica/corporativa (RAG sobre tus propios docs)
- Bug Journal: documentar causa raíz + solución de cada error encontrado
- ADRs: documentar el "por qué" de cada decisión arquitectónica importante
- Jerarquía RTFM para saber dónde buscar antes de capturar
