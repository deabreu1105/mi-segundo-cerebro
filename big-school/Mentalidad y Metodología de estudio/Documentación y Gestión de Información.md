### 2. Notas Detalladas y Captura
- **Jerarquía de la Verdad:** El orden para buscar soluciones es: 1) Documentación Oficial (RTFM - empezando por guías *Quickstart*), 2) Código fuente, 3) Libros, 4) Foros, 5) IA (al final, porque puede alucinar).
- **El Código dice QUÉ, la Documentación dice POR QUÉ:** No se debe comentar lo evidente. La documentación existe para preservar el contexto de negocio y justificar el razonamiento.
- **ADRs (Registros de Decisión de Arquitectura):** Documentos cortos para registrar decisiones técnicas. Estructura: 
  - *Contexto:* El problema a resolver.
  - *Opciones:* Pros y contras de cada alternativa.
  - *Decisión:* La solución elegida.
  - *Justificación:* Por qué se eligió esa opción.
- **Diario de Errores (Bug Journal):** Registrar siempre la causa raíz y la solución de problemas complejos para crear una biblioteca propia y no reinventar la rueda ante problemas recurrentes.
